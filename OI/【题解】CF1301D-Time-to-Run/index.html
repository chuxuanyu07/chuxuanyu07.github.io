【题解】CF1301D Time to Run

Posted on 2020-02-16 Edited on 2025-03-10 In OI

趣味简单题

题目链接：CF1301D 【Time to Run】

题意简介：

给定一个 n×mn \times mn×m 的矩阵，相邻的两个格子之间有两条方向相反的有向边，长度均为 111，从左上角开始，每次可以向左、右、上、下走一格，但是不能重复走已经走过的路（可以重复经过到过的格子，不能走出网格图），问是否可能走出一条长度为 kkk 的路径（不一定要在左上角结束），如果可以，需要输出方案。

n,m≤500n,m \le 500n,m≤500

这题一开始想，还以为是个什么欧拉路的题目，但是输出方案的方式给了我们提示…

输出方案最多 300030003000 行，每一行一个数字 f,f≤109f,f \le 10^9f,f≤109 和一个长度不超过 444 的字符串。

字符串中使用字符 L,R,U,DL,R,U,DL,R,U,D，分别代表向左、右、上、下走一步。

fff 则意为重复按照字符串所说地走 fff 次。

这个输出方法很玄学，如果是我们找到了一种走法，再去考虑输出方案，显然我们还需要分割路径的字符串，这很麻烦。

所以考虑构造一种万能走法，能够囊括 所有的长度。

注意标黑的部分，那显然我们如果能找到一种走法，使得每一条边都能被遍历，显然就是最优的了。

我们令左上角为 (1,1)(1,1)(1,1) 第一行第二个为 (1,2)(1,2)(1,2) 以此类推

我们很快可以找出一种走法，可以遍历完第一行格子之间的边，和第一行与第二行之间的所有边的走法（除了 (2,1)(2,1)(2,1) 到 (1,1)(1,1)(1,1) 一条）

这中间蓝色显然是可以重复走的 懒得画了，红色是 RRR 重复，蓝色是 DULDULDUL 的重复，黄色是一个 DDD，这样显然，第二行也可以用相同的走法，从而到达了第三行…以此类推

注意一下最后一行只能 RRR 之后 LLL 回来，不能走蓝色那种了，因为它没有下一行了，特判一下即可

最后，我们会到达 (n,1)(n,1)(n,1) ，不难发现所有 (i,1)(i,1)(i,1) 到 (i−1,1)(i-1,1)(i−1,1) 的路径都是没走过的，那我们就一直 UUU 上去即可。

那这种做法是否符合输出规范呢？

显然处理一行，我们需要的行数为 333 : 红色一行，蓝色一行，黄色一行

nmax=500n_{max} = 500nmax​=500 ，所以大约需要 150015001500 行的样子，显然 1500<30001500 < 30001500<3000，所以行数限制是没问题的。

再来看字符串长度问题，根据上面所说，我们只会走 R,DUL,D,U,LR,DUL,D,U,LR,DUL,D,U,L 这 555 种字符串，显然长度没有超过 444 ，这个限制也没有问题

再有 fff 的限制(f≤109)(f\le 10^9)(f≤109)，回想一下，我们的走法都在一行或者一列里面走，循环次数显然不会超过 max(n,m)max(n,m)max(n,m) 那肯定就不会超过 10910^9109 啦

那么我们就做完了。

注意特判 m=1m=1m=1 的情况！ 为什么？

因为我们的走法第一步就是 RRR ，但是 m=1m=1m=1 时，我们没有 RRR 的余地！

所以特判一下直接 DDD 再 UUU 就好了
